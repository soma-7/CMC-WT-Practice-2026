# Описание базы данных

<img width="469" height="303" alt="bd_commit_1" src="https://github.com/user-attachments/assets/2a4ba992-89ee-4035-ae51-2abfa37033ad" />


`Clients` - таблица клиентов, содержит jsonb-поле `info` с данными клиента (паспортные данные, водительские права, любые документы и информация, обязательная и необязательная для регистрации, контактная информация (e-mail)) и timestamp даты регистрации.

`Profiles` - таблица профилей (у одного клиента может быть несколько профилей, профиль привязывается к уникальному номеру телефона - аналогия с сим-картой) и имеет строковое имя (никнейм). jsonb-поле `other` может включать в себя дополнительную информацию профиля (например, данные карты для быстрых платежей и пополнения, ссылку на список контактов). Представляет собой json-объект со строковыми ключами и строковыми значениями. float-поле `balance` помимо хранения информации о текущем балансе служит для проверки корректности совершённых операций (оно должно быть равно сумме всех изменений баланса во всех операциях, относящихся к данному профилю).

`Operatioins` - таблица фактов, включающая в себя данные о совершённых в определённый момент времени операциях разных типов (пополнение счёта, списание со счёта, платёж за услугу, покупка услуги, отказ от услуги). Операции выполняются "от имени" профилей, чтобы, например, клиент мог настроить несколько своих номеров с разными тарифами, плата за каждый будет списываться с соответствующего баланса профиля. Операция содержит в себе float-значение изменения баланса, может относиться к какой-то услуге, если речь не идёт исключительно о пополнении счёта, а также может содержать дополнительное текстовое описание.

Услуги `Services` содержат название, описание услуги, дополнительное json-поле `includes` с "тегами" включённых в услугу "подуслуг" (например, `{"sms": "100_pieses", "internet": "10GB", "weather_forecast": "included"}`) для сортировки и фильтрации рассматриваемых тарифов. json-поле `other` со строковыми ключами и строковыми значениями содержит информацию о цене услуги (первый платёж, регулярный платёж, закрытие услуги, скидки и прочее).

# Описание страниц

<img width="1523" height="632" alt="image" src="https://github.com/user-attachments/assets/9ccdd1f1-dcd1-4e1f-a40b-c39c87b2f687" />

На каждой странице есть меню сбоку, с ссылками на выделенные жирным
страницы.

- Главная страница (ссылка на этот гитхаб, информация из данного README)

- Клиенты (профили клиентов) (список строк/таблица профилей со столбцами: id клиента, дата регистрации, баланс, телефон, ник). Можно фильтровать по балансу, сортировать по балансу и в алфавитном порядке по нику/имени. Также выполнять поиск по нику/имени/телефону. Запись в списке ведёт на страницу клиента с его профилями для просмотра и редактирования (эта страница содержит ещё кнопки удаления профилей, про удаление подробнее см. в сценариях). Кнопка для регистрации нового клиента (ведёт на страницу регистрации клиента и его профилей, можно ввести все поля кроме id, `registration_date` и баланса).

- Просмотр информации о клиенте (вся информация по всем профилям клиента, кнопка для редактирования (ведёт на страницу редактирования клиента, редактировать можно все поля, кроме id, `registration_date` и баланса).

- Услуги (список строк/таблица со столбцами: id услуги, название, описание (в таблице возможно урезано, полностью показано на странице услуги), теги и их значения (см. описание БД). Можно фильтровать и сортировать по тегам и их значениям, по названиям в алфавитном порядке. Запись ведёт на страницу услуги, на которых вся информация об услуге, кнопка для удаления услуги и кнопка для редактирования (редактировать можно все поля, кроме id). Страница услуг содержит также кнопку добавления услуги (заполнять можно все поля, кроме id).

- Операции (список строк/таблица со столбцами: id операции, id клиента, к которому относится операция, тип операции, id услуги (если есть), описание (если есть), время, изменение баланса. Фильтровать и сортировать можно по всем полям, в том числе устанавливать временной промежуток. Из записи можно перейти на страницы соответствующего клиента и услуги. Операции неизменяемы. Кнопка добавления операции (можно заполнить всё, кроме id и времени).

# Описание некоторых типовых сценариев

- Просмотр списка профилей с фильтрацией

Главная страница → Клиенты → применение фильтров, сортировка → получение отфильтрованного списка профилей.

- Просмотр истории операций по конкретному профилю за заданный интервал

Главная страница → Операции → выбор нужного профиля (через id или переход со страницы клиента) → установка временных границ (дата/время начала и конца) → получение списка операций по данному профилю за указанный период, с возможностью сортировки и фильтрации по другим полям.

- Регистрация нового клиента и создание первого профиля

Главная страница → Клиенты → кнопка регистрации нового клиента → заполнение обязательных и дополнительных данных клиента (в том числе произвольной информации в поле `info`) → добавление первого профиля (уникальный телефон, никнейм) → сохранение. Система автоматически устанавливает дату регистрации и нулевой начальный баланс.

- Пополнение счёта профиля

Главная страница → Операции → кнопка добавления операции → выбор профиля по id, выбор типа операции «Пополнение», указание суммы (положительное число) → опционально добавление описания → сохранение. Баланс выбранного профиля увеличивается на указанную сумму. Сценарий аналогичен для подключения услуги (нужно будет заполнить ещё поле id услуги).

- Удаление клиента, профиля, операции или услуги

На соответствующей странице просмотра (клиента, операции или услуги) предусмотрена кнопка удаления. При удалении клиента каскадно удаляются все его профили и связанные операции. Удаление услуги возможно только при отсутствии активных ссылок на неё в операциях. Удаление операции возможно при отсутствии профилей, на которые ссылается данная операция.
